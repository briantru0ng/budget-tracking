# Sort Transactions

A focused workflow for categorizing uncategorized and skipped transactions. Designed for speed — work through a queue of unknowns rather than scrolling through a full table.

## Merchant-Grouped Sorting

Transactions are grouped by normalized merchant name. Each group shows:

- Merchant name and sample description
- Transaction count and total amount
- Date range and source bank(s)
- Color-coded: red for expenses, green for income

For each group, pick a Basket and sub-category, then click the checkmark to apply. This also **teaches** the categorization rule so future imports of the same merchant auto-categorize.

Click the skip button to mark the entire merchant group as SKIP (excluded from all totals).

## Zelle / Venmo (Case-by-Case)

Zelle and Venmo transactions vary by person and can't be grouped by merchant. These are listed individually and must be categorized one at a time.

## Skipped Transactions

Previously skipped transactions appear in a separate section at the bottom. You can:

- Recategorize them into a real basket/category
- Move them back to uncategorized (undo skip)

## Adding New Sub-Categories

If none of the existing sub-categories fit, use the "Need a new sub-category?" section at the bottom to create one under any basket without leaving the page.
